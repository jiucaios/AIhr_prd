AI面试（补充人才画像）机器人

产品需求文档 增量画像 · 动态追问 人才画像增强模块

1. 需求背景

功能背景
虽然有AI解析简历，但是简历内容不全面，无法全方位了解人才画像，没有结构化行为特征，有些简历项目描述模糊：比如一段经历同样是“参与xx项目”但决策者和边缘执行者不应放在同一纬度比较，检索维度单一，导致后续检索时无法按“主动性”进行软素质筛选。本功能旨在通过AI面试对话机器人，追问简历缺失的画像维度，输出增量标签，并且根据简历深层次提问简历项目，确定简历中项目究竟是边缘角色还是主要角色，标签化后，提升后续人才库的可检索性。

目标与衡量指标

目标衡量指标
为每个候选人按照简历缺失程度补充画像字段按照结构化输出完整画像字段 ≥95%
面试中不重复简历已有信息人工抽检100条对话，要求命中率为0
判断真实项目贡献（根据简历项目提问细节，简历可以经过无数次修改，但临场发挥的逻辑更表现真实水平）成功输出“主导/参与/边缘”三级标签

2. 用户与场景

用户角色使用场景触发动作
候选人收到人才库补充画像的邀请邮件/短信点击链接 → 完成对话
HR/系统按照软素质多维度/标签搜索候选人进入界面勾选标签或者能力搜索

3. 功能范围

✅ In（本功能负责）
  • 接收候选人简历的json解析结果
  • 自动识别简历中缺失的结构化画像维度
  • 动态生成对话问题
  • 实时根据候选人回答触发追问
  • 每轮至少输出一个画像字段
  • 最终输出一个json格式的增量画像，写入人才库
❌ Out（不负责）
  • 不负责发送面试邀请，由上层调动
  • 不负责维护题库，可以有固定题库但题库的更新不由本功能负责

4. 用户流程

流程图示意：
候选人收到邀请 → 进入对话 → 接收简历JSON → 缺失字段识别 → 动态提问(6轮引擎) → 输出增量标签 → 写入人才库。具体流程详见下方“功能详细说明”的6轮对话逻辑。

5. 功能详细说明

轮次触发条件问题生成逻辑追问条件
1（硬性指标补全）简历中缺失的结构化中的硬性字段（如英语等级、日语等级、特定证书）且在该岗位配置中被标记为必问。系统读取解析简历的json，和结构画像进行匹配，对缺失字段进行提问。回答模糊，比如提问英语等级，回答“英文还不错，口语能对话”。
2（项目角色真实性验证）项目中有“主导”等明确项目主要地位用词生成项目的追问，比如：您当时主要职责是什么？请用一两句说明您具体完成了哪些工作。若使用模糊化用词“参与”“协助”“负责整体”，“我们”则跳转到3（项目角色深度）字数＜10触发二次追问。
3（项目角色深度）项目中有“负责”“我们”“协助”等模糊化地位词语主要围绕模糊化词语的地方进行追问，比如您负责xx项目，您的主要贡献是什么？若逻辑出现“协调部门”等问题可以提出“所以您主要是协调性工作？”字数＜10触发二次追问。
4（目标岗位与技能偏差）项目经验和目标岗位标签是否不匹配，比如开发岗项目内容是运维提问为什么之前干的运维想要来开发岗。回复“我想试试”等空泛词语，追问是否为了这个开发岗准备了什么技能和自己开发了什么项目。
5（语言组织/逻辑能力测试）始终触发描述您最近解决的一个技术性难题。无量化结果和没有按照star法则描述，语言无逻辑抓不到重点，则提示候选人有条理的重新回答，给候选人二次机会。
6（岗位特殊要求测试）按需触发我们岗位是运维岗，需要7X24小时是否能接受？；我们这个岗位是急招，需要三天内到岗是否能接受？词义匹配不追问，记录结束。

兜底策略与异常处理（通用）

异常情况判定条件兜底动作
回答过短字数 < 5 字追问：“您能再具体一点吗？一句话即可。”
明显答非所问语义相似度 < 0.3（与问题主题无关）重新表述问题：“可能我没说清楚，我想问的是......”
连续2次无效同一轮触发两次兜底仍无效跳过该维度，记录“未采集/不符合”，进入下一轮
超时无输入单题等待 > 120 秒提示：“您是否还在线？”再等10秒无响应则结束会话
检测到格式化输出（AI生成意见）同时满足多项：①典型AI句式（首先...其次...最后/综上所述）；②无第一人称“我”的具体行动；③内容与简历脱节；④回答时间极短但内容很长（<3秒输入超100字）；⑤语义流畅但无个人化细节。不打断，如果全部满足则标记为高AI生成，满足1-2项标记为低可信度AI生成，追问带个人经历问题，生成响应的标签（高AI/低AI）。
超时未输入、突然离线短线超过120s未输入记录已有信息json，下次候选人进入继续进度问答。

字段名类型必填说明示例

7. 验收标准

功能验收：

  • AC-01：系统层面正常解析发送简历和生成对应问答，第一轮中必问缺失的硬性指标，且不重复问简历中已有的技能，年限，公司等。
  • AC-02：追问必须符合要求，不允许生成无关追问，轮次符合要求，可多轮。
  • AC-03：至少预设的六轮全部覆盖。

异常处理验收：

  • AC-04：测试候选人回复无关问题答非所问的时候能够及时追问，并且记录。
  • AC-05：单轮两次无效问答后能系统记录字段跳到下一轮。
  • AC-06：输入超时（120秒后）系统及时追问，在等待一个规定时间如（15s）后自动终结会话，记录已有信息，下次从这一步中断开始。
  • AC-07：所有输出的字段能稳定输出json，并且能准确写入数据库，且原有简历不会被覆盖。
输出产物：最终生成增量画像JSON，写入人才库，增强多维检索能力（主动性、贡献度等软素质标签）。
